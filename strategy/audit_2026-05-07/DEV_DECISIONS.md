# Engineering decisions made on operator's behalf during Tier-1 sprint

> One line per decision with rationale. Appended as decisions are made.

- **R2 ElevenLabs MODULE shipped as T10, requirements.txt NOT modified** — superseded prior "deferred". Code-deliverable per brief scope (R-numbered item not in deferred list). Activation steps (signup, pip install, requirements.txt addition, config flip) remain operator-gated per `feedback_llm_api_policy.md`. Module deferred-imports `elevenlabs` so pipeline keeps working without the dep installed.
- **Caption style switch is config-driven, not CLI flag** — `config.captions.style: "word_pop"|"legacy"` cleaner than the spec's `--legacy-captions` CLI flag, fits existing pipeline.py config-dispatch idiom (cf. `tts.provider`).
- **Stage 07.5 (loudnorm) replaces inline loudnorm in `_vo_edge_tts`** — to avoid double-normalization, the inline `af=f"loudnorm=I={lufs}:LRA=11:TP=-1.5"` filter in `_vo_edge_tts` will be removed when wiring Stage 07.5. The proper two-pass takes over.
- **Postmortems land at `Channels/ShadowVerse/postmortems/`** — per Phase A Day 6.3 spec; that dir doesn't exist today; module creates it on first run.
- **Backfill of 9 existing masters is operator task, not engineering task** — Day 2.2 / Day 6.2 / Day 7.5 are operator-driven; tooling ready for them but sprint won't execute backfills.
- **Integration smoke is a discrete-tool chain, not a daily_batch.py end-to-end** — daily_batch.py requires multiple manual LLM halts (idea-gen, script-gen, fact-check, metadata-gen) and operator-only gate 3, so a true end-to-end smoke is infeasible in this autonomous sprint. Smoke instead = run new tools against existing `2026-05-07_001_master.mp4` + corrupt `2026-05-06_002_tt.mp4` and report deltas.
- **Two-pass over single-pass loudnorm (T1)** — R1 `linear=true` requires measured input stats from pass 1 for accurate target-locking; single-pass drifts ±0.5–1.0 LU on short content.
- **`MediaIntegrityError` is a hard error, not a `PipelineHalted`** — corruption is a fail-loud condition, distinct from intentional pause semantics.
- **Two-tier integrity check: ffprobe + 1-second deep-decode (T2)** — ffprobe alone misses moov-atom-present-but-stream-unreadable cases; 1s decode catches those at ~100ms cost.
- **Bundled font assumption Montserrat Black + Anton in `C:\\Windows\\Fonts` (T3)** — Phase A 3.2 makes font install operator task; module reads font name from config, fails-loud at render time if libass can't find it.
- **Active-word ASS styling via per-word Dialogue lines, not karaoke `\\k` tags (T3)** — cleaner than karaoke `\\k` (avoids separate Pop style); cost is 3-4× event count, libass handles 100+ events fine.
- **`cleanup_orphans.py` `--dry-run` is the default; live mode requires `--execute` (T4)** — inverts the spec's default to make accidental destruction impossible.
- **`archive_published.py` copies, doesn't move (T5)** — `05_exports/` stays canonical hot-tier; `06_published/` is cold-backup.
- **R6 cited-observation check is #13 in `prepublish_qa.py`, opt-in via `--check-cited-observation` (T7)** — Phase A 4.6 says checks 1-12 MVP; R6 + Day 25 says add as #13. Wired but default off until Day 25.
- **`tts.provider` config DEFAULT remains `edge-tts`, not `elevenlabs` (T8)** — overrides R2 default-flip until operator approves activation per LLM API Policy. Operator flips config flag when ready.
- **Stereo `-ac 2` enforced unconditionally in `render_master` and `_fade_variant` (T8/R7)** — current mono is the bug; no back-compat flag.
- **`prepublish_qa` Stage 11 raises `PipelineHalted` not `RuntimeError` (T8)** — failures are "fix upstream and re-run", which is halt semantics not crash semantics.
- **`youtube_upload.py` post-upload hook failures log+continue, do NOT fail the upload (T9)** — upload is the side-effecting irreversible action; bookkeeping shouldn't make operator re-upload.
- **T10 ElevenLabs WAV output 48kHz, not MP3** — matches existing chain contract; module decodes via ffmpeg.
- **T1 loudnorm target needs encode-drift compensation (open, 2026-05-11)** — Stage 7.5 two-pass loudnorm correctly lands the raw VO WAV at -14.0 LUFS, but the libx264+AAC render chain (NVENC fallback path, driver 12.2 vs req'd 13.0) drifts the final variant LUFS to ~-15.10 LU, JUST past Stage 11 check #8's ±1.0 tolerance window. Verified empirically on `_11_002` 2026-05-11 (yt -15.10, tt -15.08, ig -15.03). One-off workaround applied: post-render `ffmpeg -af "volume=+1.1dB"` bump on the 3 variants → 36/36 checks PASS (yt -13.94, tt -13.99, ig -14.01). Permanent fix options: (a) raise `config.loudnorm.target_lufs` from -14.0 to ~-12.9 so the post-encode lands at ≈-14.0; (b) add a post-render Stage 7.6 that runs loudnorm on each variant after AAC encode rather than on the raw WAV. Pick after a couple more empirical runs confirm the +1.1 LU drift is consistent across content lengths.

- **P0 Sprint 5 candidate: Stage 11 check #13 — template-artifact regex scan (open, 2026-05-13)** — `2026-05-12_002` shipped to scheduled-Private with a literal `SCRIPT_BODY (uses HOOK_A as the verbal opener):` line at the top of `script_FINAL.txt`, which edge-TTS dutifully spoke aloud. Force re-edit on match. Pair the check with a one-line reminder in `prompts/03_script_generation.md` that gate-2 stripping of separator-annotations is mandatory.
